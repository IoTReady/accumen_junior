PEAK_IPL_C_API PEAK_IPL_Library_GetLastError ( PEAK_IPL_RETURN_CODE *  lastErrorCode,
char *  lastErrorDescription,
size_t *  lastErrorDescriptionSize 
)

Queries the last error.

This function is normally used by applying a two-step procedure. First of all, call the function with all arguments except of lastErrorDescription.

// Error handling is omitted
PEAK_IPL_RETURN_CODE lastErrorCode = PEAK_IPL_RETURN_CODE_SUCCESS;
size_t size = 0;
PEAK_IPL_GetLastError(&lastErrorCode, NULL, &size);

The function then gives you the last error code and the size of the error description. You could stop now if you only want to query the last error code. If you want to query the error description as well, you have to go on.

// Error handling is omitted
char errorDescription[size];
PEAK_IPL_GetLastError(&returnCode, errorDescription, &size);

This two-step procedure may not be necessary if you just pass a buffer big enough for holding the description at the first function call.

Parameters
[out]lastErrorCodeThe last function error code.
[out]lastErrorDescriptionThe description for the last error.
[in,out]lastErrorDescriptionSizeIN: Size of the given buffer - OUT: Size of the queried data
Returns
PEAK_IPL_RETURN_CODE_SUCCESS Success
PEAK_IPL_RETURN_CODE_INVALID_ARGUMENT At least one of the arguments is a null pointer
PEAK_IPL_RETURN_CODE_BUFFER_TOO_SMALL lastErrorDescription is valid but lastErrorDescriptionSize is too small
