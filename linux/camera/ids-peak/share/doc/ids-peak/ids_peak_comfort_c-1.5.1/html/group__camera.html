◆ peak_camera_handle

typedef struct peak_camera* peak_camera_handle

peak Camera handle

The camera handle represents an opened camera.
The camera handle is provided by the peak_Camera_Open_Xxx functions and must be released by peak_Camera_Close.
The camera handle is required in every function which controls a camera of its features.

The value for an invalid camera handle is PEAK_INVALID_HANDLE.

◆ peak_camera_id

typedef uint64_t peak_camera_id

peak Camera ID

The camera id is a unique identifier for a camera.

The camera related functions which can be used without opening a camera require the camera id as identifier.
The id for a certain camera will not change over the session runtime as long as the camera is not re-connected. A re-connected camera will be assigned a new unique camera id on its re-appearance.
I.e. a once assigned camera id is considered as burnt and will not be reused for a later (re-)connected camera.

The value for an invalid camera id is PEAK_INVALID_CAMERA_ID.

◆ peak_Camera_GetAccessStatus()

peak_access_status peak_Camera_GetAccessStatus ( peak_camera_id  cameraID)

Query the camera access status.

Provides the current access status for a camera which is specified by its camera id.

This function implements the Access Status Query principle.

Parameters
[in]cameraIDThe camera id.
Returns
PEAK_ACCESS_READWRITE The camera is available for read access and for write access and can be opened.
PEAK_ACCESS_NONE The camera is not available for opening.
PEAK_ACCESS_INVALID The function failed. Call peak_Library_GetLastError to get the error code and description.

If the function indicates an error by returning PEAK_ACCESS_INVALID, these are the possible errors:

  • PEAK_STATUS_CAMERA_NOT_FOUND There is no camera with the specified ID.
  • PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
  • PEAK_STATUS_ERROR An unexpected internal error occurred.

peak_status peak_Camera_UserDefinedName_Get ( peak_camera_handle  hCam,
char *  userDefinedName,
size_t *  userDefinedNameSize 
)

Get the user defined name.

Reads the current user defined name from the camera.

This function implements the Two Staged Data Query principle.

Parameters
[in]hCamThe camera handle.
[out]userDefinedNamePointer to a user allocated character buffer to receive the cameras user defined name. If this parameter is NULL, userDefinedNameSize will contain
the length of the string, including the terminating 0.
The required size of userDefinedName in bytes is userDefinedNameSize x sizeof(char).
[in,out]userDefinedNameSize
  • userDefinedName equal NULL:
    out: length of the string, including the terminating 0
  • userDefinedName unequal NULL:
    in: number of chars userDefinedName can hold
    out: number of chars filled by the function, including the terminating 0
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_BUFFER_TOO_SMALL userDefinedName is not NULL and the value of *userDefinedNameSize is too small to receive the expected amount of data.
PEAK_STATUS_INVALID_PARAMETER userDefinedName and/or userDefinedNameSize are invalid pointers.
PEAK_STATUS_INVALID_HANDLE hCam is an invalid camera handle.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ERROR An unexpected internal error occurred.
Note
For PEAK_CAMERA_TYPE_UEYE_PLUS_U3V non-ASCII characters may be shown as _.

  • For PEAK_CAMERA_TYPE_UEYE_PLUS_GEV userDefinedName must not be larger than 16 characters, including the terminating 0.
  • For PEAK_CAMERA_TYPE_UEYE_USB and PEAK_CAMERA_TYPE_UEYE_ETH the function accepts numeric strings only, where the numeric value is in the range (1,254).

◆ peak_CameraList_Get()

peak_status peak_CameraList_Get ( peak_camera_descriptor *  cameraList,
size_t *  cameraCount 
)

Query the current camera list.

Provides the current list of detected cameras.

This function implements the Two Staged Data Query principle.

Parameters
[out]cameraListPointer to a user allocated array buffer to receive the camera list. If this parameter is NULL, cameraCount will contain the current number of cameras.
The needed size of cameraList in bytes is cameraCount x sizeof(peak_camera_descriptor).
[in,out]cameraCount
  • cameraList equal NULL:
    out: minimal number of cameras cameraList must be large enough to hold
  • cameraList unequal NULL:
    in: number of cameras cameraList can hold
    out: number of cameras filled by the function
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_BUFFER_TOO_SMALL cameraList is not NULL and the value of *cameraCount is too small to receive the expected amount of data.
PEAK_STATUS_INVALID_PARAMETER cameraCount is an invalid pointer.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ERROR An unexpected internal error occurred.
Note
Consider that the camera list might change between the size query call and the list query call. This may be the case if peak_CameraList_Update is called and cameras have been attached or detached in the time between the two function calls. To eliminate this issue you may want to use an array for cameraList which is large enough to hold all possibly connected cameras and to spare the size query call.

◆ peak_CameraList_Update()

peak_status peak_CameraList_Update ( size_t *  cameraCount)

Update the camera list.

Initiates a camera discovery and an update of the camera list.

This function is required to be called to create or update the camera list. The camera list does not change unless this function is called again.

Parameters
[out]cameraCountThe number of connected cameras. NULL if not required.
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ERROR An unexpected internal error occurred.
Note
Depending on the installed transport layers the camera list may take some hundred milliseconds. This is because the discovery of cameras via so called connectionless network protocols like ethernet requires a timeout controlled procedure.
