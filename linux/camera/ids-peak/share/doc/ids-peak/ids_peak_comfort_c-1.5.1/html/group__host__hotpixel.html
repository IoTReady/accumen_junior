◆ peak_hotpixel_correction_sensitivity

enum peak_hotpixel_correction_sensitivity

peak hotpixel correction sensitivity level

Higher sensitivity levels mean more hotpixels will be detected and corrected, but can also lead to more false-positives.

Enumerator
PEAK_HOTPIXEL_CORRECTION_SENSITIVITY_INVALID 

Invalid hotpixel correction sensitivity level.

Use this value for the initialization of variables of type peak_hotpixel_correction_sensitivity.

PEAK_HOTPIXEL_CORRECTION_SENSITIVITY_LEVEL_1 

hotpixel correction sensitivity level 1

This is the lowest sensitivity level, detecting the fewest hotpixels, therefore may miss more hotpixels, creating more false-negatives.

PEAK_HOTPIXEL_CORRECTION_SENSITIVITY_LEVEL_2 

hotpixel correction sensitivity level 2

PEAK_HOTPIXEL_CORRECTION_SENSITIVITY_LEVEL_3 

hotpixel correction sensitivity level 3

This is the default sensitivity level.

PEAK_HOTPIXEL_CORRECTION_SENSITIVITY_LEVEL_4 

hotpixel correction sensitivity level 4

PEAK_HOTPIXEL_CORRECTION_SENSITIVITY_LEVEL_5 

hotpixel correction sensitivity level 5

This is the highest sensitivity level, detecting the most hotpixels, but may create more false-positives.

peak_status peak_IPL_HotpixelCorrection_GetList ( peak_camera_handle  hCam,
peak_position *  hotpixelList,
size_t *  hotpixelCount 
)

Get the list of detected host hotpixel correction positions/pixels.

Queries the list of detected host hotpixel correction positions/pixels.

This function implements the Two Staged Data Query principle.

Parameters
[in]hCamThe camera handle.
[out]hotpixelListPointer to a user allocated array buffer to receive the hotpixel positions list.
If this parameter is NULL, hotpixelCount will contain the current number of hotpixel positions.
The required size of hotpixelList in bytes is hotpixelCount x sizeof(peak_position).
[in,out]hotpixelCount
  • hotpixelList equal NULL:
    out: minimal number of hotpixel positions hotpixelList must be large enough to hold
  • hotpixelList unequal NULL:
    in: number of hotpixel positions hotpixelList can hold
    out: number of hotpixel positions filled by the function
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_BUFFER_TOO_SMALL hotpixelList is not NULL and the value of *hotpixelCount is too small to receive the expected amount of data.
PEAK_STATUS_INVALID_PARAMETER hotpixelCount is an invalid pointer.
PEAK_STATUS_INVALID_HANDLE hCam is an invalid camera handle.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ERROR An unexpected internal error occurred.
Note
Please consider that the hotpixel positions list might change between the size query call and the list query call.
This may be the case if the camera configuration or the camera status have changed in the time between the two function calls.

peak_status peak_IPL_HotpixelCorrection_ResetList ( peak_camera_handle  hCam)

Reset the list of host hotpixel positions.

Resets the list of hotpixel positions that should be corrected, so they are automatically detected again.

By default, the host hotpixel correction automatically detects the hotpixel positions once and uses these positions for the hotpixel correction in the following images. This function resets/clears these positions, so they are automatically detected again.

Returns
