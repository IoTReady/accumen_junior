IDS peak comfortC  v1.5.1
ChangeLog

List of changes in each version

v1.5.1

  • Speed up initial device list update for Ethernet cameras.

v1.5

  • Add new functions for a Message Queue.
  • Add new functions for I2C.

v1.4

  • Add new functions for running inferences.
  • Add new function peak_IPL_Sharpness_Measure for measuring sharpness of an image.
  • Add new functions for host side edge enhancement
  • Fix user provided hotpixel list getting overwritten on first call to peak_IPL_ProcessFrame or peak_IPL_ProcessFrameInplace.
  • Fix user provided hotpixel list getting reset when calling peak_IPL_HotpixelCorrection_Sensitivity_Set or changing the region of interest
  • Fix hotpixel list passed to peak_IPL_HotpixelCorrection_SetList as non-const pointer.
  • Fix peak_camera_descriptor.userDefinedName not updated if the name has been changed. The name will now be updated when calling peak_CameraList_Update.
  • Breaking changes
    • peak_IPL_HotpixelCorrection_Sensitivity_Set will no longer reset the hotpixel list, if provided by the user
    • The hotpixel list will no longer be reset when the region of interest changes, if the list was manually provided by the user

v1.3

  • Add access to the user defined name property of a camera.
  • Add new functions for controlling the active focus and auto focus.
  • Add new functions peak_Camera_UserDefinedName_Set and peak_Camera_UserDefinedName_Get to control the user defined name of a camera.
  • Add new function peak_Camera_ID_FromUserDefinedName to enable opening a camera by its user defined name.
  • Add the user defined name to the peak_camera_descriptor.
  • Add the property saturation to the host color correction feature.
    • Add new functions peak_IPL_ColorCorrection_Saturation_Get, peak_IPL_ColorCorrection_Saturation_Set, and peak_IPL_ColorCorrection_Saturation_GetRange.
  • Fix possible use after free if the process is closed and peak_Library_Exit has not been called.
  • Fix an issue where peak_IPL_AutoBrightness_TargetTolerance_Set sets the target value instead of the tolerance value
  • Fix an issue where peak_Binning_Set and peak_Decimation_Set returns PEAK_STATUS_SUCCESS instead of PEAK_STATUS_OUT_OF_RANGE
  • Fix an issue where peak_PixelFormat_GetList reports non-selectable pixelformats

v1.2

  • Add new Video functions to encode frames and append them inside a video file
  • Add new CameraMemory functions to access file storage on cameras
  • Fix an issue where peak_ColorCorrection_GetAccessStatus for uEye camera models returns PEAK_ACCESS_READWRITE instead of PEAK_ACCESS_READONLY
  • Fix an issue where peak_IOChannel_Xxx_GetAccessStatus returns PEAK_ACCESS_INVALID instead of PEAK_ACCESS_NOT_SUPPORTED for not supported io channels
  • Fix an issue where peak_Acquisition_Start would return an error when starting the acquisition a second time in manual buffer management mode if buffers are not revoked between acquisitions

v1.1

  • Add functionality to save and load .ini parameterset files from uEye to peak_CameraSettings_DiskFile_Store and peak_CameraSettings_DiskFile_Apply.
  • Add new functions peak_IPL_AutoBrightness_ROI_Offset_GetRange and peak_IPL_AutoBrightness_ROI_Size_GetRange to query the current range of valid values for the host auto brightness ROI offset and dimensions
  • Add new functions peak_IPL_AutoWhiteBalance_ROI_Offset_GetRange and peak_IPL_AutoWhiteBalance_ROI_Size_GetRange to query the current range of valid values for the auto white balance ROI offset and dimensions
  • Add PEAK_STATUS_VALUE_ADJUSTED as return value to the function peak_PixelClock_Set to address an issue for uEye camera models with a discrete list of valid pixel clock values.
  • Add new cmake function ids_peak_comfort_c_generate_starter_script to generate a .sh script to start your application, which will take care of setting LD_LIBRARY_PATH, GENICAM_GENTL32_PATH and GENICAM_GENTL64_PATH, especially when using the .tar.gz package
  • The cmake function peak_windeploy has been marked as deprecated and replaced by ids_peak_comfort_c_deploy to resolve naming conflicts with generic SDK packages
  • Fix an issue where the C99 standard would not be set automatically when using newer cmake versions (>= 3.8)
  • Fix an issue where peak_IPL_ColorCorrection_Matrix_Set did not work as expected
  • Fix an issue where all peak_AutoBrightness_ROI_Xxx and peak_AutoWhiteBalance_ROI_Xxx functions return PEAK_STATUS_OUT_OF_RANGE instead of PEAK_STATUS_ACCESS_DENIED
  • Fix an issue where peak_GFA_Enumeration_GetList reports all existing entries and not just the selectable ones
  • Fix an issue where peak_Trigger_Mode_Set returned PEAK_STATUS_OUT_OF_RANGE instead of PEAK_STATUS_ACCESS_DENIED
  • Fix an issue where peak_IPL_AutoBrightness_TargetPercentile_Set did not work as expected
  • Fix an issue with the internal image buffer management in Manual Buffer Preparation mode
  • Breaking changes
    • peak_Acquisition_Stop always returns PEAK_STATUS_SUCCESS now, even if no acquisition is running.
    • peak_Acquisition_Stop will now block until parallel stop calls (or the automatic stop in case of a finite acquisition sequence) have finished

v1.0
