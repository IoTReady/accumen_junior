Stores the most significant byte of a word at the smallest memory address and the least significant byte at the largest

Stores the least-significant byte at the smallest address.

Query the decimation access status.

Provides the current access status for the decimation feature.

This function implements the Access Status Query principle.

Parameters
[in]hCamThe camera handle.
Returns
PEAK_ACCESS_READWRITE The decimation feature is readable and writeable.
PEAK_ACCESS_READONLY The decimation feature is readable only.
PEAK_ACCESS_GFA_LOCK The decimation feature is not accessible because the GFA write access is enabled.
PEAK_ACCESS_NONE The decimation feature is not accessible.
PEAK_ACCESS_NOT_SUPPORTED The decimation feature is not supported.
PEAK_ACCESS_INVALID The function failed. Call peak_Library_GetLastError to get the error code and description.

If the function indicates an error by returning PEAK_ACCESS_INVALID, these are the possible errors:

  • PEAK_STATUS_INVALID_HANDLE hCam is an invalid camera handle.
  • PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
  • PEAK_STATUS_ERROR An unexpected internal error occurred.

Measures the sharpness in the image.

Measures the sharpness in the image or image roi using the specified sharpness algorithm. The measured values are relative so you should compare values at different focus levels to find the best sharpness value. Higher values correspond to sharper images.

[in]roiThe roi in which to measure the sharpness. Must be at least 20x20 pixels.
[in]sharpnessAlgorithmThe algorithm used for measuring sharpness.
[out]calculatedValueThe measured sharpness value.
