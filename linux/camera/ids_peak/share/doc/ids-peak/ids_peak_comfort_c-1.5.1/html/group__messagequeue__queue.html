◆ peak_MessageQueue_EnabledMessages_GetList()

peak_status peak_MessageQueue_EnabledMessages_GetList ( peak_message_queue_handle  hMessageQueue,
peak_message_type *  messageTypesArray,
size_t *  messageTypesArraySize 
)

Query the current message types enabled for the queue.

Provides the current list of message types enabled for the queue.

This function implements the Two Staged Data Query principle.

Parameters
[in]hMessageQueueHandle for a message queue instance.
[out]messageTypesArrayPointer to a user allocated array buffer to receive the message type list. If this parameter is NULL, messageTypesArraySize will contain the current number of enabled message types.
The needed size of messageTypesArray in bytes is messageTypesArraySize x sizeof(peak_message_type).
[in,out]messageTypesArraySize
  • messageTypesArray equal NULL:
    out: minimal number of message types messageTypesArray must be large enough to hold
  • messageTypesArray unequal NULL:
    in: number of message types messageTypesArray can hold
    out: number of message types filled by the function
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_INVALID_HANDLE hMessageQueue is an invalid handle.
PEAK_STATUS_BUFFER_TOO_SMALL messageTypesArray is not NULL and the value of *messageTypesArraySize is too small to receive the expected amount of data.
PEAK_STATUS_INVALID_PARAMETER messageTypesArraySize is an invalid pointer.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ERROR An unexpected internal error occurred.
Note
Consider that the list might change between the size query call and the list query call if message types are enabled or disabled between the two calls. To eliminate this issue you may want to use an array for messageTypesArray which is large enough to hold all possibly types and to spare the size query call.

◆ peak_MessageQueue_WaitForMessage()

peak_status peak_MessageQueue_WaitForMessage ( peak_message_queue_handle  hMessageQueue,
uint32_t  timeout_ms,
peak_message_handle *  hMessage 
)

Wait for a new message.

A waiting call can be aborted by calling peak_MessageQueue_Stop.

The function may return with PEAK_STATUS_WARNING_OVERFLOW. It indicates an overflow of the queue and messages were discarded. It is advisable to increase the call frequency to peak_MessageQueue_WaitForMessage, increase the queue size or decrease the message frequency.

Note
After processing the message, it must be freed by a call to peak_Message_Release.
Parameters
[in]hMessageQueueHandle for a message queue instance.
[in]timeout_msThe wait timeout in milliseconds. PEAK_INFINITE for infinite.
[out]hMessagePointer to a message handle receiving the message instance.
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_WARNING_OVERFLOW Operation was successful; messages were dropped because the queue is overflowing.
PEAK_STATUS_INVALID_HANDLE hMessageQueue is an invalid handle.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ACCESS_DENIED The queue is stopped.
PEAK_STATUS_ABORTED The queue was stopped while waiting.
PEAK_STATUS_NO_DATA No data received.
PEAK_STATUS_TIMEOUT The timeout was reached with no message.
PEAK_STATUS_ERROR An unexpected internal error occurred.
