◆ peak_PixelFormat_GetAccessStatus()

peak_access_status peak_PixelFormat_GetAccessStatus ( peak_camera_handle  hCam)

Query the pixel format access status.

Provides the current access status for the pixel format feature.

This function implements the Access Status Query principle.

Parameters
[in]hCamThe camera handle.
Returns
PEAK_ACCESS_READWRITE The pixel format feature is readable and writeable.
PEAK_ACCESS_READONLY The pixel format feature is readable only.
PEAK_ACCESS_GFA_LOCK The pixel format feature is not accessible because the GFA write access is enabled.
PEAK_ACCESS_NONE The pixel format feature is not accessible.
PEAK_ACCESS_NOT_SUPPORTED The pixel format feature is not supported.
PEAK_ACCESS_INVALID The function failed. Call peak_Library_GetLastError to get the error code and description.

If the function indicates an error by returning PEAK_ACCESS_INVALID, these are the possible errors:

  • PEAK_STATUS_INVALID_HANDLE hCam is an invalid camera handle.
  • PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
  • PEAK_STATUS_ERROR An unexpected internal error occurred.

◆ peak_PixelFormat_GetList()

peak_status peak_PixelFormat_GetList ( peak_camera_handle  hCam,
peak_pixel_format *  pixelFormatList,
size_t *  pixelFormatCount 
)

Get the list of currently selectable pixel formats.

Queries the list of currently selectable pixel formats.

The list of selectable pixel formats may depend on the camera configuration and the camera status.

This function implements the Two Staged Data Query principle.

Parameters
[in]hCamThe camera handle.
[out]pixelFormatListPointer to a user allocated array buffer to receive the pixel format list. If this parameter is NULL, pixelFormatCount will contain the current number of pixel formats.
The required size of pixelFormatList in bytes is pixelFormatCount x sizeof(peak_pixel_format).
[in,out]pixelFormatCount
  • pixelFormatList equal NULL:
    out: minimal number of pixel formats pixelFormatList must be large enough to hold
  • pixelFormatList unequal NULL:
    in: number of pixel formats pixelFormatList can hold
    out: number of pixel formats filled by the function
Returns
PEAK_STATUS_SUCCESS Operation was successful; no error occurred.
PEAK_STATUS_BUFFER_TOO_SMALL pixelFormatList is not NULL and the value of *pixelFormatCount is too small to receive the expected amount of data.
PEAK_STATUS_ACCESS_DENIED The pixel format feature is not accessible. Check the access status of the pixel format feature via peak_PixelFormat_GetAccessStatus.
PEAK_STATUS_INVALID_PARAMETER pixelFormatCount is an invalid pointer.
PEAK_STATUS_INVALID_HANDLE hCam is an invalid camera handle.
PEAK_STATUS_NOT_INITIALIZED The library is not initialized.
PEAK_STATUS_ERROR An unexpected internal error occurred.
Note
Consider that the pixel format list might change between the size query call and the list query call.
This may be the case if the camera configuration or the camera status have changed in the time between the two function calls.
